# ReviewPing — Project Tracking

## Goal
Stabilize ReviewPing as production-ready SaaS with full SEO public pages and SPA routing on Vercel.

## Constraints & Preferences
- Vite + React only (no Next.js/Express)
- Supabase Edge Functions for backend
- Inline styles with G theme (no Tailwind)
- $19/mo alternative to Podium — target restaurants, clinics, salons, e-commerce
- No `window.alert()` / `window.confirm()` — use `ConfirmModal` + `sonner` toasts
- No mock/seed data — every data point from Supabase
- All public pages must have Google-optimized meta tags, JSON-LD schema, and content
- Deploy via: `npm run build && node scripts/prepare-vercel-output.mjs && npx vercel deploy --prebuilt --prod`
- Vercel Git auto-deploy: **DISABLED** (causes middleware conflict with SPA routing)
- Vercel framework: **null (Other)** — using Build Output API v3 directly

## Blocks Removed
- ✅ **SPA routing fix (was P0):** Solved by switching to manual prebuilt deployment via Build Output API v3. Vercel's Vite framework auto-generates a middleware lambda that breaks SPA routing. By using `framework: null` + custom `.vercel/output/config.json` with rewrite rules + `vercel deploy --prebuilt`, all routes work correctly.
